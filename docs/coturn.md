# Coturn Setup Notes

TURN is required for users behind restrictive NATs or firewalls. STUN-only setups will fail for some real users.

## Minimal Coturn Configuration

```text
listening-port=3478
fingerprint
lt-cred-mech
realm=your-domain.com
user=TURN_USERNAME:TURN_PASSWORD
no-multicast-peers
no-cli
```

Use TLS-enabled TURN in production and expose both UDP and TCP relay paths.

## Client ICE Server Example

Pass this into `useCall({ iceServers })` or `new CallManager({ iceServers })`:

```js
const iceServers = [
  { urls: 'stun:stun.l.google.com:19302' },
  {
    urls: 'turn:turn.your-domain.com:3478',
    username: process.env.EXPO_PUBLIC_TURN_USERNAME,
    credential: process.env.EXPO_PUBLIC_TURN_PASSWORD,
  },
];
```

For production systems, prefer short-lived TURN credentials generated by your backend instead of shipping static credentials in the client.
